If you prefer to use AI in your terminal, Claude Code CLI is one of the recommended tools for you. We don't provide paid licenses for Claude Code CLI to use their model directly. You are only allowed to use Claude Code CLI with AWS Bedrock models provided by HelloFresh.

Pre-RequisitesÂ¶

* Access to AWS Bedrock account (bedrock1). Here you can find the instructions on how to get access.

* AWS CLI installed. To install AWS CLI, follow the instructions.

* Basic knowledge of using the command line.

InstallationÂ¶

Follow the official documentation provided by Claude Code to install and configure Claude Code CLI. You can follow the recommended steps below to install Claude Code CLI in order to use HelloFresh provided AWS Bedrock models.

Step 1: Install Claude Code CLIÂ¶

Follow the official installation instructions from Claude to install the CLI. After installation, you can run claude --version to check if the installation was successful. It would return the version of the Claude Code.

Step 2: Configure AWS CredentialsÂ¶

There are multiple ways to configure AWS credentials. You can follow the official documentation to configure your AWS credentials from Claude Code documentation.

However, we prefer to setup a SSO profile for AWS for making it easy to re-authenticate when needed.

To setup SSO profile,

1. Make sure you have $HOME/.aws/config file exists.

2. Add the following AWS Bedrock SSO profile to the $HOME/.aws/config file for accessing bedrock1 AWS Bedrock account.

# existing profiles





[profile sso-bedrock]

sso_start_url = https://hfsso.awsapps.com/start

sso_region = eu-west-1

sso_account_id = 951719175506

sso_role_name = bedrock-user

region = eu-west-1





	3. 

Verify that the profile is working by running aws sso login --profile sso-bedrock. After successful login, run aws sts get-caller-identity --profile sso-bedrock to verify that you are able to access AWS Bedrock account.

Step 3: Configure Claude Code CLI to use AWS Bedrock modelsÂ¶

   1. Make sure you have $HOME/.claude/settings.json file exists. If not, create this file.

   2. Add the following configuration to the $HOME/.claude/settings.json file.

{

    "$schema": "https://json.schemastore.org/claude-code-settings.json",

    "awsAuthRefresh": "aws sso login --profile sso-bedrock",

    "env": {

      "AWS_PROFILE": "sso-bedrock",

      "AWS_REGION": "eu-west-1",

      "CLAUDE_CODE_USE_BEDROCK": "1",

      "ANTHROPIC_MODEL": "eu.anthropic.claude-sonnet-4-5-20250929-v1:0"

    }

}





	Important

   * Your model ID must be adjusted if you use different region. For example, eu.anthropic.claude-sonnet-4-5-20250929-v1:0 is for eu-west-1 region.

   * You can find the list of available models here.

Step 4: Test your installationÂ¶

Open your terminal and run claude "HI" to see Claude Code CLI in action. If it responds with an answer, you are ready to use Claude Code CLI. You would see it in action like the below:

FAQÂ¶

   * Do I need to login to Claude Code CLI to use it?

No. Because we don't provide paid licenses for Claude Code CLI to use their model directly. Instead, we use AWS Bedrock models provided by HelloFresh. Therefore, you don't need to login to Claude Code CLI to use it. Also, you are not allowed to use Claude Code CLI with your personal licenses.

      * Can I use any other model with Claude Code CLI?

Yes, you can use any models that are available in AWS Bedrock in "bedrock1" AWS Bedrock account. You can find the list of available models here.

         * I see errors "Too many requests" or related to rate limiting errors when I run Claude Code CLI. What should I do?

It's a known issue mostly happen due to the rate limiting of AWS Bedrock models. Please try again later. We continuously monitor quotas of AWS Bedrock models and will request the quota if needed to AWS. If you still see the issue, please reach out to the #clan-genai or the #squad-engineering-productivity.
