1. Goal





Our goal is to create a single, unified prospects checkout layout with standardized features that will work centrally for all Mealkit brands (HelloFresh, GreenChef, EveryPlate, and ChefsPlate) and RTE brands (Factor and Youfoodz).[a][b][c][d][e]





Checkout is a critical phase in the acquisition funnel where users register or log in, provide their address, select delivery options, and add payment details.

Unified feature is a shared component that can operate across multiple countries, brands, and user journeys. Once implemented, any new functionality built on this Feature is automatically available wherever the feature is deployed, ensuring consistency across the board. Features list are available on the portal: https://www-staging.hellofresh.com/features?f=DeliverySummaryFeature

Modular checkout is the checkout code setup, which consists of  independent, reusable unified Features.

Checkout layout is a structure for how elements are interconnected as part of it.





RTE - Ready-to-Eat Brands (Factor, Youufoodz)

WLMK/New Mealkits - Mealkit brands (GreenChef, EveryPlate, and ChefsPlate)









2. Problem Statement





The HelloFresh Group has expanded to 8 brands across 18 countries, which requires scalable solutions to avoid isolated work efforts. The prospects checkout process to purchase subscription can be identical across all brands and products, which makes it the first user experience that can be standardized and managed centrally.





In January 2025, the activation user journey scope was transferred from TAM Alliance to Growth Alliance to centralize Checkout ownership within one team. Two brands, GoodChop and Pets Table, remain in TAM. Scope now increased to 31 distinct codes to manage (18 in HelloFresh with 6 in New Mealkits and 7 in RTE added on top). To scale further and grow sustainably, a strong foundation is needed. HelloFresh has set an 2025 objective #1 to accelerate product development velocity for Ready-to-eat (RTE) brands by distributing ownership of core customer journeys across the consumer-facing-alliances (CFAs).





The Reduce Effort Needed to Checkout strategy initiated in 2024 involved refactoring the HelloFresh checkout into a modular checkout with unified features. By the end of 2024, the scope expanded to include RTE and WLML, aiming to unify features across all meal brands. More details are in Modular Checkout. The goal for H1-2025 is to complete feature modularization and unify the layouts for all meal brands. The new checkout layout will use the technical format that was created during the Single Page Checkout initiative. More details in Single Page Checkout: Experiment Results.
